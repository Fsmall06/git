# ESP-111 与 `esp-idf` / `esp-csi` 的调度映射

快照日期：2026-07-10。以下结论只基于当前工作区实时源码和
`findings.md` 中固定 commit 的一手源码证据；未做编译、真机负载或时延测试。

分类含义：

- **direct-borrow**：外部模式与本地约束一致，可直接保留或按同一原则扩展。
- **experiment**：方向合理，但必须先做核占用、时延、队列水位或硬件压力实验。
- **do-not-copy**：示例代码适合 bring-up，却会破坏 ESP-111 的生产边界或所有权规则。

证据等级沿用 `findings.md`：A 为固定 commit 源码直接证明；本地证据均为当前实时源码直接证明。

## 可立即进入主文档的结论

1. C5 的真正约束是单核总预算；dispatcher/worker 拆分正确，但不是核隔离。
2. ESP-111 当前最成熟的可复用点是“callback/handler 短路径 + 有界队列 + 明确 ownership + drop/latency 指标”。
3. 最值得做真机实验的两处是 C5 phase-level WDT 和 S3 core affinity；两者都不能仅凭源码静态判断收益。
4. 最明确的禁止项是 callback 整帧打印、pointer queue 投递失败不释放、以及在重复网络事件内无幂等地创建 producer task。
